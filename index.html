📅 Monthly Claim Limits (rolling 30-day window)

Control how many times each person can receive a control number within a rolling 30-day window. Set to ♾️ Unlimited to disable the limit. These apply independently per person.

👤
Claimant Monthly Limit
How many times a claimant (matched by name) can receive a number within 30 days
Claimants with the same name cannot claim more than this many times in a rolling 30-day window.
♾️ Claimants can claim unlimited times per month.
🧑‍🤝‍🧑
Beneficiary 1 Monthly Limit
How many times Beneficiary 1 (matched by name or ID) can be registered within 30 days
Beneficiary 1 with the same name or ID cannot be claimed more than this many times in a 30-day window.
♾️ Beneficiary 1 can be claimed unlimited times per month.
🧑‍🤝‍🧑
Beneficiary 2 Monthly Limit
How many times Beneficiary 2 (matched by name or ID) can be registered within 30 days
Beneficiary 2 with the same name or ID cannot be claimed more than this many times in a 30-day window.
♾️ Beneficiary 2 can be claimed unlimited times per month.
📌 How the 30-day window works:
The system counts all tickets from the past 30 days up to the target distribution date. If a person already has tickets equal to or exceeding the limit, the new request is blocked and a cooldown message is shown. The existing 7-day cooldown still applies in addition to this monthly limit. Setting a limit to ♾️ Unlimited disables the monthly check entirely for that person type.
